Principios y conceptos avanzados de la POO

1. Principios Fundamentales

La Programación Orientada a Objetos se basa en:

  • Encapsulamiento: Ocultar los detalles internos de un objeto.
  • Abstracción: Mostrar solo la información esencial.
  • Herencia: Reutilizar código mediante clases derivadas.
  • Polimorfismo: Permitir que una misma acción tenga diferentes comportamientos.

2. Clases abstractas

Una clase abstracta no se puede instanciar directamente. Define métodos que deben ser implementados en las subclases.

class Figura:
    def area(self):
        raise NotImplementedError

class Circulo(Figura):
    def __init__(self, radio):
        self.radio = radio

    def area(self):
        return 3.14 * self.radio ** 2

3. Interfaces (simuladas en Python)

Una interfaz define un conjunto de métodos sin implementación. Se asegura de que todas las clases que la implementen tengan esos métodos.

class Conectable:
    def conectar(self):
        pass

class USB(Conectable):
    def conectar(self):
        print("Conectado por USB")

4. Métodos y atributos estáticos

Los métodos estáticos pertenecen a la clase, no a la instancia. Se usan con @staticmethod o @classmethod.

class Util:
    contador = 0  # Atributo estático

    @staticmethod
    def saludar():
        print("Hola desde Util")

    @classmethod
    def incrementar(cls):
        cls.contador += 1

5. Niveles de acceso: públicos, protegidos y privados

En Python se simulan con convenciones:

  • publico: sin guiones bajos (acceso libre).
  • _protegido: un guión bajo (solo dentro de la clase y subclases).
  • __privado: dos guiones bajos (oculto fuera de la clase).
class Persona:
    def __init__(self):
        self.nombre = "Ana"        # Público
        self._edad = 30            # Protegido
        self.__dni = "1234567890"  # Privado
