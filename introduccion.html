Introducción a la Programación Orientada a Objetos

¿Qué es la Programación Orientada a Objetos?

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el diseño de software en torno a objetos, entidades que contienen datos y comportamientos. Este enfoque permite estructurar programas de manera que sean más modulares, reutilizables y fáciles de mantener. La POO se basa en la idea de modelar el software de manera similar a cómo percibimos el mundo real, donde los objetos interactúan entre sí.

Conceptos Básicos de la POO

Clase

Una clase es una plantilla o modelo que define las características y comportamientos que tendrán los objetos de ese tipo. Funciona como un plano que especifica qué atributos (datos) y métodos (funciones) tendrán los objetos creados a partir de ella. Las clases permiten encapsular datos y comportamientos relacionados en una única entidad.

Objeto

Un objeto es una instancia específica de una clase. Mientras que la clase es el modelo teórico, un objeto es una entidad concreta creada a partir de ese modelo, con valores específicos para sus atributos. Cada objeto tiene su propio estado interno (valores de sus atributos) pero comparte el comportamiento (métodos) definido por su clase.

En este ejemplo, vemos cómo a partir de una única clase Estudiante se pueden crear múltiples objetos, cada uno con sus propios valores de atributos. Todos los objetos comparten la misma estructura y comportamientos definidos por la clase, pero cada instancia mantiene su propio estado.

Podemos observar que los atributos (como id, nombre, apellido) almacenan datos específicos de cada estudiante, mientras que los métodos (como getNombreCompleto, estaAprobado) definen comportamientos que pueden ser invocados sobre cualquier objeto de tipo Estudiante.

Implementación en Python

class Estudiante:
    """
    Clase que representa a un estudiante con sus atributos y métodos.
    """
    
    def __init__(self, id, nombre, apellido, promedio, activo=True):
        """Constructor que inicializa los atributos del estudiante."""
        self.id = id
        self.nombre = nombre
        self.apellido = apellido
        self.promedio = promedio
        self.activo = activo
    
    def get_nombre_completo(self):
        """Retorna el nombre completo del estudiante."""
        return f"{self.nombre} {self.apellido}"
    
    def esta_aprobado(self):
        """Verifica si el estudiante está aprobado (promedio >= 7.0)."""
        return self.promedio >= 7.0
    
    def actualizar_promedio(self, nota):
        """Actualiza el promedio del estudiante con una nueva nota."""
        self.promedio = (self.promedio + nota) / 2
        return self.promedio
    
    def cambiar_estado(self, estado):
        """Actualiza el estado de actividad del estudiante."""
        self.activo = estado
        return self.activo


# Creación de los objetos (instancias) de la clase Estudiante
estudiante1 = Estudiante(1001, "Ana", "García", 9.2)
estudiante2 = Estudiante(1002, "Carlos", "Pérez", 7.8)
estudiante3 = Estudiante(1003, "Laura", "Martínez", 8.5, False)

# Uso de los métodos en los objetos
print(f"{estudiante1.get_nombre_completo()} está aprobado: {estudiante1.esta_aprobado()}")
print(f"{estudiante2.get_nombre_completo()} está aprobado: {estudiante2.esta_aprobado()}")
print(f"{estudiante3.get_nombre_completo()} está aprobado: {estudiante3.esta_aprobado()}")

# Actualización del promedio de un estudiante
nuevo_promedio = estudiante2.actualizar_promedio(6.0)
print(f"Nuevo promedio de {estudiante2.get_nombre_completo()}: {nuevo_promedio}")
print(f"{estudiante2.get_nombre_completo()} sigue aprobado: {estudiante2.esta_aprobado()}")

# Cambio de estado de un estudiante
estudiante1.cambiar_estado(False)
print(f"{estudiante1.get_nombre_completo()} está activo: {estudiante1.activo}")

Este código Python muestra la implementación práctica de la clase Estudiante con sus atributos y métodos correspondientes. Podemos ver cómo se crean las tres instancias (objetos) con diferentes valores y cómo cada objeto mantiene su propio estado mientras comparte los comportamientos definidos por la clase.

También se ilustra el uso de los métodos para obtener información (get_nombre_completo, esta_aprobado) y modificar el estado de los objetos (actualizar_promedio, cambiar_estado).
